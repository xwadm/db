## 高优先级任务
### 底层云适配：补齐引擎启动兼容逻辑
部分数据库引擎无法使用标准`spindb start`指令启动，云端部署脚本采用绕过方案处理，补齐功能后可统一启动流程，减少版本差异。

- [x] **QuestDB 启动超时优化**：该数据库基于Java虚拟机，冷启动耗时60-120秒，原程序固定60秒超时、Windows端90秒超时。现已上调至通用120秒、Windows150秒。镜像更新后，云端可废弃Java直接启动的临时方案。
- [ ] **MariaDB 自定义配置文件支持**：当前启动参数禁用默认配置，无法加载云端所需TLS配置文件，需新增配置文件路径指定能力。
- [x] **MongoDB 身份验证参数**：支持`spindb start 实例名 --auth`指令，自动开启数据库鉴权，配置持久化存储。
- [x] **FerretDB 鉴权重启适配**：2.0版本启动时不再强制关闭鉴权，支持SCRAM安全认证模式，鉴权状态可留存配置。
- [ ] **SurrealDB 凭据重置功能**：云端场景需清空数据目录重置账号密码，当前工具未对外开放该操作接口。
- [x] **InfluxDB 预加载管理员令牌**：本地启动与数据恢复流程可读取留存的管理员令牌文件，完善权限校验体系。
- [ ] **Weaviate 集群状态重置与环境变量注入**：云端需清空集群数据状态、启动时注入自定义环境变量，现有功能暂不支持。
- [x] **CockroachDB 本地安全模式**：单个实例独立生成安全证书，基于证书目录启动服务，支持带TLS加密的查询、备份与恢复操作。

- [ ] **静默创建跳过工具校验**：SQLite、DuckDB这类文件型数据库，执行`spindb create --no-start`会提前校验客户端工具，此时尚未下载程序包导致创建失败。静默创建无需校验运行工具，该逻辑异常会影响云端镜像预制流程，临时替代方案为手动下载引擎程序。
- [ ] **品牌LOGO与视觉素材定稿**：梳理素材目录内图标设计方案，敲定托盘图标、应用标识、文字LOGO样式。
- [ ] **全引擎Docker导出测试**：验证21款数据库均可正常打包为容器镜像，包含文件型数据库，确保镜像启动、连接访问、数据持久化全部可用。
- [ ] **二进制包下载源统一管理**：集中配置数据库安装包的下载地址，统一管控源地址规则。
- [ ] **Windows系统WSL代理层**：封装适配层，让Windows设备无缝调用Linux子系统内数据库服务。
- [ ] **二进制包自主托管迁移**：脱离第三方依赖库，将数据库编译包迁移至项目自有服务存储。
- [ ] **正向/反向代理基础版本开发**：搭建网络转发体系，支持容器网络对外访问部署。
- [ ] **Git钩子联动**：接入提交、切换分支等钩子事件，自动执行数据库同步、环境重置等流程。
- [ ] **应用接口优化**：梳理精简程序调用接口，稳定对外调用规范。
- [ ] **封装可调用程序包**：将命令行能力封装为函数包，支持代码层面直接调用。
- [ ] **测试体系重构**：优化测试用例执行逻辑，缩减运行耗时。
- [ ] **统一数据库默认账号密码**：规范所有引擎初始登录凭据标准。

## 版本迭代路线
### v1.1 远程连接与密钥管理
- [x] **外部数据库关联**：通过连接字符串绑定远端数据库，自动识别数据库类型与服务厂商；支持连接、地址查询、信息查看、列表管理、删除、数据查询操作；凭据加密存储；后续迭代新增远端备份、脚本执行能力。
- [ ] **环境变量读取适配**：连接字符串可引用系统环境变量，支持变量形式创建、接入数据库。
- [ ] **密钥安全存储**：当前凭据以明文文件存储，仅基础权限防护。
  - 短期方案：基于设备标识+用户密钥加密凭据文件，消除明文存储风险，无需额外依赖组件。
  - 长期方案：跨设备凭据同步，设备密钥配对授权，云端加密存储，本地私钥解密读取。
  - 备选方案：调用系统密钥管理器，受限于原生依赖，暂不符合当前编译部署架构。

## 技术债务整改
### 严重问题
#### 多版本程序调用错乱
安装同一引擎多个版本后，实例可能调用错误版本程序，引发运行异常
- [ ] 校验版本匹配逻辑，绑定实例与对应程序路径
- [ ] 启动前置校验程序版本，和实例配置保持一致
- [ ] 优化多版本程序缓存调用规则

#### 第三方下载地址失效风险
PostgreSQL固定下载链接会随官方页面更新失效
- [ ] 动态抓取最新版本地址，摒弃硬编码链接
- [ ] 远端配置文件兜底，实时同步版本映射关系
- [ ] 下载前置校验链接可用性
- [ ] 定时监测官方新版本发布

#### Windows解压命令注入风险
调用系统命令解压文件存在安全隐患
- [ ] 替换系统命令，改用原生文件操作接口
- [ ] 校验文件路径合法性，限定操作目录范围

#### 输入内容安全校验
用户自定义名称可能注入脚本命令，存在执行风险
- [ ] 规范实例、数据库命名格式，拦截特殊字符
- [ ] 调整进程调用方式，规避字符串拼接执行漏洞

### 中等问题
#### SQLite路径校验缺失
指定不存在目录创建数据库仍可提交，失败后无回滚处理
- [ ] 创建前校验上级目录合法性
- [ ] 新增事务机制，异常自动撤销操作
- [ ] 支持自动创建目录参数

#### 多进程并发文件损坏
多个命令行同时操作配置文件，易造成数据损毁
- [ ] 新增文件互斥锁机制
- [ ] 改用数据库存储配置，提升并发操作稳定性

#### 版本校验层级错误
非法版本号不会即时拦截，后续下载才报错，提示模糊
- [ ] 命令层提前校验版本合法性
- [ ] 统一各类引擎错误提示文案

### 轻微优化项
#### 进度展示不完善
- [ ] 划分下载、解压、校验、启动等流程阶段
- [ ] 展示文件操作百分比进度
- [ ] 长耗时备份还原实时反馈状态

#### MongoDB备份格式兼容不全
暂不支持单备份文件自动识别集合名称

#### 超长脚本执行限制
大批量SQL语句通过命令行传参存在长度上限
- [ ] 改用标准输入流传入脚本内容，规避系统限制
